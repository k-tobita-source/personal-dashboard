# 個人用カンバンダッシュボード 要件叩き台

---

## 1. 目的・背景

当日の予定、Slack の依頼・確認事項、Gmail、継続的な ToDo がバラバラに管理されており、整理・優先順位付けに手間がかかっている。これらを **1画面のカンバンボード** に集約し、「対応が必要なタスク」を俯瞰・管理する。あわせて Claude（Claude Code / Claude Desktop）から **MCP サーバー経由でボードを操作**（追加・整理）できるようにする。

## 2. スコープ

### 2.1 やること
- 3レーン構成のカンバンボード（Web UI）
- Google カレンダー / Slack メンション / Gmail / 独自 ToDo の集約取得
- カードのレーン間移動による状態管理
- ドラッグ&ドロップによる予定化・タスク化
- ローカルDB
- ボード操作を提供する MCP サーバー

### 2.2 やらないこと
- クラウド保存・外部同期 / マルチユーザー・共有 / 他人の予定編集 / モバイルアプリ化

## 3. 最重要の非機能要件

- **ローカル完結**：外部 API 取得も状態保存もローカル PC 内で行い、第三者サーバーへ送信しない。
- **最小権限**：Google = 予定の閲覧＋作成のみ、Slack = 検索のみ、Gmail = 閲覧のみ。
- **認証情報のローカル保管**：トークン類はローカルにのみ保存し、`.gitignore` でリポジトリ除外。

---

## 4. 機能要件：カンバンボード

### 4.1 レーン構成

| レーン | 内容 | 備考 |
|---|---|---|
| Schedule | Google カレンダー / Slack メンション / Gmail / 独自 ToDo を時系列表示 | 「現在時刻ライン」を表示。時刻未定カードは現在時刻直下の「未割当」帯にまとめる |
| In Progress | 対応中のカード | |
| Done | 完了したカード | |

- **状態の単一ソース**：カードの所属レーンを状態の唯一の真実とする。別途の状態ラベルは持たず、レーンが状態を兼ねる。

### 4.2 Schedule レーンの挙動
- 時刻確定（カレンダー予定・時刻指定 ToDo）はタイムライン上の該当位置へ配置。
- 時刻未定（Slack メンション・Gmail・無期限 ToDo）は「未割当」帯にまとめる。
- 現在時刻ラインを表示し、進行中の予定をハイライト。

### 4.3 ドラッグ&ドロップ
- 未割当カードをタイムラインへドラッグ → 時刻が決まりスナップ配置。
- Slack メンション / Gmail を予定化・ToDo化できる。

### 4.4 外部ソースの扱い分け
- カレンダー予定・Slack メンション・Gmail は「外部実体」。Done に移してもボード上の状態のみ変わり、元データは削除・変更しない。
- 独自 ToDo はボード内で完結する自前データ。
