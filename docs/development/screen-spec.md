# 個人用カンバンダッシュボード 画面仕様

> [`spec.md`](./spec.md)（要件）と認識合わせ（ASCIIワイヤー）の結果を、実装・DB設計の入力としてまとめたもの。
> 技術選定は [`tech-stack.md`](./tech-stack.md) を参照。

---

## 1. 全体レイアウト：4カラム

spec の「3レーン＋未割当帯」を、**受信箱 ＋ 3レーン** の4カラム構成に再構成する。

```
┌──────────────────────────────────────────────────────────────────────────────────┐
│  my-kanban                                          2026-06-10 (火) 14:30  [更新⟳]  │
├─────────────────┬──────────────────────┬───────────────────┬───────────────────────┤
│ 📥 受信箱        │ 🗓 Schedule           │ ▶ In Progress     │ ✓ Done                │
│ (時刻未定)       │ (タイムライン)         │                   │                       │
├─────────────────┼──────────────────────┼───────────────────┼───────────────────────┤
│ 時刻未定カード    │ 09:00 📅 朝会         │ 対応中のカード     │ 完了カード             │
│ を縦に列挙        │ 11:00 📅 設計MTG      │ （積み上げ）       │ （積み上げ）           │
│                 │ ━ 14:30 ◀ 現在 ━━━━━ │                   │                       │
│                 │ 16:00 📅 1on1         │                   │                       │
└─────────────────┴──────────────────────┴───────────────────┴───────────────────────┘
  凡例: 📅カレンダー  💬Slack  📧Gmail  ✅ToDo                          [+ ToDo追加]
```

| カラム | 役割 | 並び |
|---|---|---|
| 📥 受信箱 | 時刻未定の取り込みカード（Slackメンション / Gmail / 無期限ToDo） | カード積み上げ |
| 🗓 Schedule | 時刻確定カード（カレンダー予定 / 時刻指定ToDo）を時間軸上に配置 | **当日 0:00〜24:00 の縦タイムライン**（時刻位置に配置） |
| ▶ In Progress | 対応中のカード | カード積み上げ |
| ✓ Done | 完了したカード | カード積み上げ |

- **状態の単一ソース**：カードの所属カラムが状態の唯一の真実（spec §4.1）。別途の状態ラベルは持たない。

---

## 2. Schedule カラム（タイムライン）

- **表示範囲**：当日 0:00〜24:00 を縦スクロール。初期表示位置は現在時刻付近。
- **現在時刻ライン**：`━ HH:MM ◀ 現在 ━` のラインを表示し、進行中の予定をハイライト（spec §4.2）。
- **配置**：時刻確定カードを開始時刻に応じてタイムライン上へ配置。
- 過去の予定も上方向にスクロールすれば閲覧可能（自動で消さない）。

---

## 3. 受信箱カラム

- 時刻未定の外部実体（Slackメンション・Gmail）と無期限ToDoをまとめる。
- ここを起点に Schedule / In Progress へドラッグして処理する。

---

## 4. ドラッグ&ドロップ（spec §4.3）

| 操作 | 結果 |
|---|---|
| 受信箱 → Schedule のタイムラインへドロップ | ドロップ位置の時刻にスナップし、**時刻が確定** |
| 受信箱 → In Progress へドロップ | 時刻未定のまま**着手**状態に |
| 任意のカラム → Done | 完了。外部実体は元データを変更しない（ボード状態のみ） |

- **【決定】D&D で決めた時刻を実 Google カレンダーへは書き込まない**（[`tech-stack.md`](./tech-stack.md) §4.5）。

---

## 5. カードの表示項目

| 項目 | 内容 | 備考 |
|---|---|---|
| ソースアイコン + タイトル | 📅/💬/📧/✅ と件名・要約 | 全カード必須 |
| 本文プレビュー / 送信者 | Slack本文の冒頭、Gmail送信者・抹要など | 外部実体カードで表示。独自ToDoは無し |
| 元ソースへのリンク | 元の Slackスレッド / Gmail / カレンダー予定を開く | 外部実体カードで表示 |

- **日時（開始・期限）はカード上に文字表示しない**。タイムライン上の配置で時間を表現する。
  ※ ただしデータとしては保持する（タイムライン配置・並び替えに必須）。

---

## 6. カードの種別

| 種別 | 例 | Done に移したとき |
|---|---|---|
| 外部実体 | カレンダー予定 / Slackメンション / Gmail | ボード状態のみ変更。**元データは削除・変更しない**（spec §4.4） |
| 独自ToDo | 自分で追加したタスク | ボード内で完結（自前データ） |

---

## 7. 未確定・後続検討

- 受信箱 → In Progress へ直接移した時刻未定カードの、Schedule との関係（着手中に時刻を付与できるか等）。
- 受信箱の並び順（新着順 / ソース別 など）。
- カラム幅の比率（Schedule を広めにする等）。
